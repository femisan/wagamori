## Where to comment (targeting)
Prioritize commenting on posts that **already have high engagement — ~100+ likes**, not small/low-traffic posts. Post genuine **appreciation or a precise, on-point observation/感嘆** about that specific post.

**Why:** The user set this strategy explicitly. High-like posts have many active viewers, so a comment there exposes the Wagamori brand name to far more people → more profile visits/反流. Commenting on tiny posts (e.g. ~90-follower accounts, single-digit likes) wastes the effort.

**How to apply:** When building a comment batch, filter candidate posts by like count (check via `instagram user <handle> --limit N` → `likes` field) and favor 100+. Likes and follows can still target small/mid relevant accounts broadly; this 100+ rule is specifically for COMMENTS. Keep comments conservative and value-adding — genuine praise or a sharp specific remark, never generic. The user also prefers to preview new comments before they go out (treat as default until told otherwise).

## How to write (style)
When posting comments (and engagement replies), **do not use many emoji / 顔文字** — heavy emoji is a giveaway that reads as AI-generated. Keep comments emoji-light: 0, or at most 1 subtle one when it genuinely adds warmth, never one at the end of every line.

**Why:** The user explicitly flagged that the first comment drafts (each ending with an emoji like 🕊️🍋🌊🩷✨) "特别像是AI生成" — too many emoji makes the brand's outreach look automated, which hurts trust and risks looking spammy.

**How to apply:** For every comment/reply written via opencli, prefer plain, specific, slightly casual Japanese that references the actual post; strip decorative emoji. Also avoid other AI tells: over-polished/effusive praise, formulaic "〜ですね。〜ますね。" stacking, and identical structure across comments — vary phrasing and let some be short.
